# 문서 최신화/정리 작업 계획

이 문서는 `docs/` 전체를 최신 개발 현황에 맞춰 정리하기 위한 작업 계획입니다.
병렬로 진행 가능한 작업 흐름을 기준으로 작성했습니다.

---

## 목표

- 현재 동작과 맞지 않는 문서를 제거하거나 최신화
- 중복/산재된 문서를 통합해 사용자 동선을 단순화
- 핵심 문서(사용자/운영/개발/아키텍처) 우선 정비

## 비목표

- 제품 기능 설계 변경
- 코드 리팩터링
- 신규 기능 추가

---

## 작업 범위

- `docs/` 전체
- 우선순위: `guides/`, `getting-started/`, `architecture/`, `INDEX.md`
- 보조: `new_whitepaper/`, `api/`, `templates/`

---

## 병렬 작업 스트림

### 스트림 A: 문서 인벤토리/분류
- 목적: 유지/통합/삭제 후보 분류
- 산출물: 문서 목록 + 분류 테이블
- 기준:
  - 최신 동작과 일치하는가
  - 사용자 여정에 직접 필요성이 있는가
  - 중복/부분 중복인가

### 스트림 B: 통합/정리 설계
- 목적: 통합 대상 문서 구조 설계
- 산출물: 통합 문서 맵
- 기준:
  - 단일 주제는 하나의 문서로 유지
  - 운영/개발/사용자 관점 분리

### 스트림 C: 핵심 문서 최신화
- 대상:
  - `docs/guides/USER_GUIDE.md`
  - `docs/guides/DEV_GUIDE.md`
  - `docs/guides/RAG_CLI_WORKFLOW_TEMPLATES.md`
  - `docs/INDEX.md`
- 산출물: 최신화된 핵심 문서

### 스트림 D: 삭제/이관 작업
- 목적: 구버전/계획/로그성 문서 정리
- 산출물: 삭제 목록 및 통합 결과 반영

---

## 수행 단계 (병렬화 가능)

1. 문서 인벤토리 및 분류 (스트림 A)
2. 통합 설계 및 대상 선정 (스트림 B)
3. 핵심 문서 최신화 (스트림 C)
4. 구버전/계획/로그 문서 정리 (스트림 D)
5. 문서 인덱스 및 링크 정비 (`docs/INDEX.md`)

---

## 통합/삭제 후보 기준

### 통합 후보
- 유사 주제(예: RAG 개선/진단/개선 로그 등)
- 운영/사용법이 중복된 문서
- 계획 문서와 구현 문서가 분리되어 있는 경우

### 삭제 후보
- 과거 작업 로그/계획서 성격 문서
- 현재 동작과 불일치한 문서
- 장기 미갱신 상태의 MVP/플랜 문서
