# 방법론 1: 폴더 지형도 + 책임 태깅

> Audience: 개발자/기여자/아키텍트
> Purpose: 디렉터리 구조를 빠르게 스캔하고 책임 경계를 태깅하는 절차를 제공
> Last Updated: 2026-01-06

---

## 핵심 질문

- “이 폴더는 무엇을 책임지나?”

---

## 사용 목적

- 온보딩 초기에 전체 지형을 빠르게 파악
- 도메인/어댑터/테스트의 책임 경계를 대략 설정
- 다른 방법론(C4/의존성 그래프)을 적용하기 전 방향 잡기

---

## 빠른 절차

1. `src/evalvault` 최상위 폴더를 목록화한다.
2. 각 폴더의 책임 키워드를 1~2개로 태깅한다.
3. 책임이 불명확한 폴더는 “검증 필요”로 표시한다.
4. 결과를 표 또는 간단한 다이어그램으로 남긴다.

---

## 짧은 예시 (개념 파악용)

```text
src/evalvault/
  domain/    # 핵심 도메인 로직
  ports/     # 계약(인터페이스)
  adapters/  # 외부 통합
  config/    # 런타임 설정
```

---

## 다른 방법론 대비 장점/단점

| 구분 | 내용 |
|---|---|
| 장점 | 의존성 그래프보다 훨씬 빠르게 전체 윤곽을 잡을 수 있다. |
| 단점 | 엔트리포인트 흐름이나 실제 실행 경로는 드러나지 않는다. |

---

## 시각화/도구

- Mermaid flowchart로 상위 폴더 맵을 만든다.
- `rg --files src/evalvault`로 실제 파일 분포를 확인한다.
- `tree -L 2`로 폴더 깊이를 제한해 요약한다.
