# Web UI 개선 백로그 (Analysis Lab 중심)

> 작성일: 2026-01-09
> 상태: v0.1 (초안)
> 목적: 실행 안정성/가시성/전문가 관점을 반영한 개선 작업을 우선순위로 정리한다.

## P0 (즉시 영향/신뢰성)

- **결과 스키마 가드 (완료)**: `final_output`/`node_results` 런타임 검증 및 에러 대비 렌더링.
  - 검증: 스키마 누락 시 UI 경고 배너 + 대체 렌더링 확인.
- **Contract 테스트 (완료)**: API 응답 스냅샷 기반 계약 테스트 추가.
  - 검증: CI에서 스냅샷 비교로 깨짐 탐지.
- **에러 가시성 강화 (완료)**: 노드 실패 원인 요약 카드 + 오류 집계 카운트.
  - 검증: 실패 노드 1개 이상일 때 경고/카운트 표시.
- **실행 요청 재현성 (완료)**: 실행에 사용된 파라미터(쿼리, 프로필, LLM 옵션)를 결과 카드에 표시.
  - 검증: 저장/로드 시 파라미터가 동일하게 노출.

## P1 (인지 부하 감소/탐색성)

- **실행 단계 타임라인 (완료)**: DAG 실행 단계/순서 시각화.
  - 검증: 노드 상태 변화가 한눈에 보이는지 사용자 테스트.
- **결과 비교 흐름 강화 (완료)**: Analysis Lab 결과 선택 → 바로 비교로 이어지는 CTA.
  - 검증: 클릭 수 2회 이내로 비교 화면 도달.
- **결과 요약 카드 확장 (완료)**: 주요 지표/실패 유형 상위 3개 요약 카드.
  - 검증: 결과 페이지 이탈률 감소(사용성 테스트).
- **대용량 출력 최적화 (완료)**: 보고서/RAW JSON 페이징/가상화 분리.
  - 검증: 10k+ 텍스트에서도 60fps 스크롤 확보.

## P2 (전문가 관점/정교화)

- **인지심리/교육공학 관점**: 안내 흐름(학습 목표, 단계적 안내, 피드백 루프) 강화.
  - 검증: 신규 사용자 onboarding 시간 감소.
- **사회심리/마케팅 관점**: 협업/공유/성과 리포트 템플릿 제공.
  - 검증: 공유 횟수/리포트 생성률 상승.
- **데이터 분석 관점**: 지표별 드릴다운/분포 시각화 탭 추가.
  - 검증: 문제 원인 파악 시간 단축.
- **UI/UX 프레임워크 관점**: 2025 트렌드 기반 레이아웃/타이포/모션 리디자인.
  - 검증: UI 일관성/정보 전달 만족도 개선.
- **예측 니즈 대응 (완료)**: 다음 액션 패널로 저장/재실행/비교 흐름 안내.
  - 검증: 다음 행동 전환율 상승.

## 우선순위 가이드

- **P0**는 반드시 계약 테스트 + 에러 복원력 확보가 먼저.
- **P1**은 분석 결과 해석 시간을 줄이는 방향으로 설계.
- **P2**는 사용자군 확장/전문성 강화 목적.
